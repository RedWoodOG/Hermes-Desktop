# Hermes Desktop v2.5.1

**Release date:** 2026-04-27  
**Portable asset:** `HermesDesktop-portable-x64.zip` (from [GitHub Releases](https://github.com/RedWoodOG/Hermes-Desktop/releases))

## Highlights

### MCP host

- **Startup bootstrap:** Hermes now loads configured MCP servers at app startup and registers discovered tools beside native tools.
- **Standard config paths:** Hermes checks `%LOCALAPPDATA%\hermes\hermes-cs\mcp.json`, `%LOCALAPPDATA%\hermes\mcp.json`, `%APPDATA%\Hermes\mcp.json`, and `%USERPROFILE%\.hermes\mcp.json`.
- **Native schemas:** MCP tools expose their server-provided input schema to the model instead of a generic `arguments` wrapper.
- **Backwards compatibility:** older MCP calls shaped as `{ "arguments": { ... } }` still unwrap correctly.

### Documentation

- Added `docs/mcp.md` with config locations, a filesystem server example, and tool naming notes.

### Engineering

- Added `McpBootstrap` to centralize MCP config loading, server connection, and tool registration.
- Added tests for schema-backed tools and MCP argument passthrough.
- Versioning: `HermesDesktop` assembly and `Package.appxmanifest` identity set to **2.5.1.0**.

## Upgrade notes

- Existing users do not need to change configuration.
- To enable MCP, add an `mcp.json` file in one of the documented config locations and restart Hermes.
