# Hermes Desktop v2.5.0

**Release date:** 2026-04-27  
**Portable asset:** `HermesDesktop-portable-x64.zip` (from [GitHub Releases](https://github.com/RedWoodOG/Hermes-Desktop/releases))

## Highlights

### Streaming reliability

- **30-second watchdog:** stalled provider streams now surface a structured timeout instead of leaving chat hung indefinitely.
- **Structured error codes:** provider failures are classified as `ProviderTimeout`, `ProviderAuth`, `RateLimit`, `Transport`, or `StreamParseError`.
- **Transport surfacing:** OpenAI-compatible and Anthropic streams now report HTTP, timeout, IO, and malformed JSON failures through `StreamEvent.StreamError`.

### Chat UX

- **Error banner:** stream failures show a visible chat banner rather than only writing a transcript/system message.
- **Recovery actions:** the banner includes **Retry** and **Switch model** actions so users can recover without hunting through Settings.
- **Localized labels:** banner actions are available in `en-us` and `zh-cn` resources.

### Engineering

- **Unit tests:** added a watchdog regression test that verifies an idle stream yields `ProviderTimeout` before late tokens arrive.
- **Versioning:** `HermesDesktop` assembly and `Package.appxmanifest` identity set to **2.5.0.0**.

## Upgrade notes

- **Portable zip:** quit the app, extract the new zip over (or beside) the old folder; keep `%LOCALAPPDATA%\hermes` -- your config, memory, transcripts, and wiki stay there.
- **Provider errors:** if a model stream stalls or a provider returns malformed stream chunks, Hermes now displays an error banner within the watchdog window. Use **Retry** for transient failures or **Switch model** to move the conversation to another configured provider.
